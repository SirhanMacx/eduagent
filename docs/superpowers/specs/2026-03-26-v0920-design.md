# Claw-ED v0.9.20 — Design Spec

## Vision

Teachers deserve an AI partner that knows their curriculum, generates beautiful materials, and gets better every day. v0.9.20 transforms Claw-ED from an agent that generates content into one that *understands* a teacher's entire body of work — and acts on that understanding.

---

## 1. Custom Agent Naming

Teachers name their agent during onboarding. The agent refers to itself by that name in all interactions.

### Changes

- `AppConfig.agent_name: str = "Claw-ED"` — new field, persisted in config.json
- `build_system_prompt()` — replaces hardcoded "You are Claw-ED" with `f"You are {agent_name}"`
- `configure_profile` tool — gains `agent_name` parameter
- Onboarding flow — after teacher shares their name, agent asks: "What would you like to call me?"
- All transports pull agent name from config for welcome messages / headers

### Files

- `models.py` — add field to AppConfig
- `prompt.py` — parameterize agent name
- `core.py` — pass agent_name through context
- `agent_core/tools/configure_profile.py` — accept agent_name param
- `onboarding.py` — add naming step

## 2. Curriculum Knowledge Base

The headline feature. Teacher files become a searchable semantic database the agent consults before every generation.

### New Module: `clawed/agent_core/memory/curriculum_kb.py`

```
CurriculumKB
├── SQLite: ~/.eduagent/memory/curriculum_kb.db
├── Table: chunks
│   ├── id INTEGER PRIMARY KEY
│   ├── teacher_id TEXT NOT NULL
│   ├── doc_title TEXT NOT NULL
│   ├── source_path TEXT
│   ├── chunk_text TEXT NOT NULL
│   ├── chunk_hash TEXT NOT NULL (for dedup)
│   ├── embedding TEXT NOT NULL
│   ├── metadata TEXT DEFAULT '{}' (subject, grade, topic tags)
│   └── created_at TEXT NOT NULL
├── Index: idx_chunks_teacher ON (teacher_id)
├── Index: idx_chunks_hash ON (teacher_id, chunk_hash)
│
├── index(teacher_id, doc_title, source_path, full_text, metadata)
│   → chunk at ~500 tokens with 50-token overlap
│   → embed each chunk via get_embedder()
│   → deduplicate by chunk_hash
│
├── search(teacher_id, query, top_k=10) → list[ChunkResult]
│   → embed query
│   → SQL: SELECT ... WHERE teacher_id=? ORDER BY created_at DESC LIMIT 200
│   → cosine similarity ranking in Python
│   → return with source attribution
│
└── stats(teacher_id) → {doc_count, chunk_count, subjects}
```

### New Tool: `search_my_materials`

- LLM description: "Search the teacher's uploaded curriculum files for relevant content. Use BEFORE generating to ground output in the teacher's own work."
- Returns ranked chunks with source: "From 'Civil War Unit 2024.docx' (p.3): ..."

### Ingest Pipeline Update

- `handlers/ingest.py` — after persona extraction, chunk and index into CurriculumKB
- Idempotent: deduplicates by source_path + chunk_hash
- Returns enriched response: "Added [file] to your curriculum library. You now have [N] documents."

### Memory Loader Update

- `memory/loader.py` — new `curriculum_context` key in returned dict
- When loading context for prompt, auto-search KB with current message as query
- Inject top results into system prompt under `## Your Curriculum Knowledge Base`

### Episodic Memory Scaling Fix

- `episodes.py` `recall()` — add `WHERE created_at > date('now', '-90 days')` and `LIMIT 200` to SQL query before Python-side cosine ranking

## 3. Beautiful Student-Facing Materials

Every export looks like it came from a curriculum publisher.

### New Module: `clawed/export_templates.py`

```python
@dataclass
class DocxTheme:
    title_font: str = "Calibri"
    title_size: int = 20  # pt
    heading_font: str = "Calibri"
    heading_size: int = 14
    body_font: str = "Calibri"
    body_size: int = 11
    accent_color: str = "#2B579A"  # Professional blue
    light_accent: str = "#D6E4F0"  # Light blue for shading
    header_bg: str = "#2B579A"
    footer_text: str = "Created with Claw-ED"

@dataclass
class PptxTheme:
    title_font: str = "Calibri"
    title_size: int = 36
    subtitle_size: int = 18
    body_font: str = "Calibri"
    body_size: int = 20
    accent_color: str = "#2B579A"
    bg_color: str = "#FFFFFF"
    slide_width: float = 13.333  # inches (16:9)
    slide_height: float = 7.5

PROFESSIONAL = {"docx": DocxTheme(), "pptx": PptxTheme()}
```

### DOCX Improvements (exporter.py)

- Professional header: teacher name, school, subject, date
- Footer: page numbers + small branding
- Consistent Calibri typography with proper heading hierarchy
- Worksheet formatting: numbered items, answer lines, point values in brackets
- Assessment formatting: bold question numbers, indented choices, answer key on separate page
- Differentiation callout boxes with shaded background for IEP/ELL
- Rubric as formatted table with shaded header row

### PPTX Improvements

- Title slide: lesson title large, teacher name + date + subject subtitle
- Section divider slides between lesson phases (Do Now, Instruction, Practice, Exit Ticket)
- Professional blue/gray palette
- Content slides: proper bullet hierarchy, not walls of text
- Speaker notes from lesson plan teacher-facing content
- Image placeholder slides: "[Add Image: topic]"

### Post-Generation Export Guarantee (loop.py)

- After agent loop completes, inspect `all_files`
- If a generation tool was called (generate_lesson, generate_unit, generate_materials) but export_document was NOT called, inject a synthetic export_document call
- Teachers always get files, even if the LLM forgets

## 5. Agentic Personality

The agent feels like a colleague, not a tool.

### System Prompt Rewrite (prompt.py)

The prompt is restructured around three behaviors:

**Search-first:** "Before generating ANY content, search the teacher's curriculum knowledge base. Reference what they've already created by name."

**Status narration:** "As you work through tasks, give brief one-sentence progress updates: 'Searching your files...', 'Found 3 related lessons. Generating now.', 'Exporting to PPTX...'"

**Proactive suggestions:** "After completing a task, suggest 1-2 natural next steps. If you made a lesson but there's no worksheet, offer. If a unit has no assessment, mention it. Be helpful, not pushy."

### Identity

```
You are {agent_name}, a teaching partner for {teacher_name}.
You are not a chatbot. You are a colleague who happens to live inside a computer.
You know {teacher_name}'s curriculum because you've read their files.
You don't wait to be told every step — you take initiative.
```

### Loop Enhancement (loop.py)

- Track which generation tools were called during the loop
- Post-generation export guarantee (Section 3)
- Store suggestion context in session for next-turn awareness

### Ingest Narration (handlers/ingest.py)

Rich response after file ingestion:
"Got it — I've added [filename] to your curriculum library. I now have [N] documents across [subjects]. This helps me reference your actual materials when I create new content."

## 6. Infrastructure Cleanup & Bug Fixes

### _LLMClientAdapter Cleanup (core.py)

Replace monkey-patching of global `TOOL_DEFINITIONS` with a clean adapter that passes tool schemas directly to API call functions. No more global state mutation.

### Configurable Max Iterations

- `AppConfig.max_agent_iterations: int = 20`
- Passed through `core.py` → `run_agent_loop()`
- Complex curriculum planning can go deeper

### Approval Tracker → SQLite (autonomy.py)

- New `~/.eduagent/approvals.db` with `approvals` table
- `get_rates()` becomes a single SQL GROUP BY query
- Migration: auto-import existing JSON files on first load
- `_NEVER_AUTO_APPROVE` set unchanged

### Web Dashboard Basic Auth

- `AppConfig.dashboard_password: Optional[str] = None`
- If set, `clawed serve` requires HTTP basic auth
- Simple guard, not full RBAC (that's v1.0)

### Telegram Direct Handlers

- Wire 14 registered slash commands to direct handlers bypassing agent loop
- `/model`, `/config`, `/export`, `/help`, `/status` etc. get instant response
- Non-command messages still route through the agent

### Code Cleanup

- Consistent import ordering across all files
- Remove dead imports and unused variables
- Docstrings on all public functions
- Type hints on all function signatures
- Remove legacy backward-compat shims that are no longer needed
- Ensure all files use `from __future__ import annotations`
